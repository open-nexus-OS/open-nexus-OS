# i18n

The DSL uses key-based i18n.

Authoring formats are human-friendly; runtime formats are deterministic and compact.

## Authoring

Example catalog (illustrative):

```json
{
  "home.title": "Home",
  "home.cta": "Continue"
}
```

## Runtime

- preferred runtime artifact: compiled `.lc` (Cap'n Proto encoded)
- JSON is an authoring/debug view, not the canonical runtime contract

## In DSL

```nx
Text { value: @t("home.title") }
Button { label: @t("home.cta") }
```

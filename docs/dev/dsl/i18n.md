# i18n

The DSL uses key-based i18n. Authoring formats are human-friendly; runtime
formats are deterministic and compact. Contract: RFC-0077.

## Authoring

One flat JSON object per locale under the app folder: `i18n/<tag>.json`
(`tag` = 1–16 ASCII alphanumerics, e.g. `en`, `de`):

```json
{
  "home.title": "Home",
  "home.cta": "Continue"
}
```

`i18n/en.json` is the DEFAULT catalog: its texts are BAKED into the program
at compile time (`@t("home.title")` resolves through the NXIR `i18nKeys`
table to the display-text symbol — the dotted key names are NOT recoverable
at runtime). Templates use `{0}`-style placeholders.

## In DSL

```nx
Text { value: @t("home.title") }
Button { label: @t("home.cta") }
```

## Bundle payload (RFC-0077)

`compile_project_bundle` (used by the bundle build) compiles every
`i18n/<tag>.json` into an index-aligned **`NXL1` locale pack** (key order =
NXIR `i18nKeys`; absent keys fall back to the baked text) and ships the
program as an **`NXLC` payload container** (NXIR + packs, tags sorted,
deterministic bytes). Apps without catalogs keep the raw `.nxir` payload —
consumers sniff the magic.

## Runtime switch

app-host splits the container at mount (fail-closed: a malformed pack only
loses that locale, never the program) and resolves `@t()` through
`active catalog → baked default`. The `ui.locale` setting travels
settingsd → windowd (`OP_WATCH`) → `OP_SURFACE_REGION` push → catalog swap +
`view.reemit()`. Selection is exact tag, then primary language subtag
(`de-DE` → `de`). Proof markers: `SELFTEST: i18n switch ok`,
`apphost: locale <tag> applied` (bounded, tag only).

Legacy note: the `.nxc`/`NXC1` binary catalog (nx-dsl CLI `i18n compile`)
remains a host-side authoring/debug artifact; shipped apps use packs.

# Developer Foundations

Foundations define the cross-cutting rules and mental models that developer-facing platform surfaces should share.

Use this category for:

- development posture and local workstation rules,
- Console and shell expectations,
- package-management and environment expectations,
- Dev Studio workflow posture,
- and other developer-facing system foundations that are broader than a single technology page.

Current entry points:

- `docs/dev/foundations/development/README.md`
- `docs/dev/foundations/development/console.md`
- `docs/dev/foundations/development/package-manager.md`
- `docs/dev/foundations/development/dev-studio.md`

Rule of thumb:

- if the question is "what are the platform-wide rules for developing locally on Open Nexus OS?", start here,
- if the question is "which product-like platform capability should my app use?", start in `docs/dev/technologies/`,
- and if the question is "how is the system internally implemented?", move to architecture, security, ADRs, or packaging docs.

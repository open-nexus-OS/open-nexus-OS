# Platform Maps

This section collects platform-level maps and keystone views that connect multiple subsystems.

Use it when you want to understand:

- how several developer-facing or system-facing areas fit together,
- which keystone pieces make a larger platform story credible,
- and where a feature sits in the broader Open Nexus OS product and architecture direction.

Current entry points:

- `docs/dev/platform/developer-workstation.md`
- `docs/dev/platform/group-and-device-management.md`
- `docs/dev/platform/keystone-map.md`
- `docs/dev/platform/qemu-virtio-mmio-modern.md`

Rule of thumb:

- if a page is broader than one developer technology,
- but still not a full architecture contract or ADR,
- it probably belongs here.

# Navigation

Navigation docs cover how users move between sections, routes, panes, and contexts.

Use this category for:

- sidebars and tab bars,
- breadcrumbs/path control,
- search-first navigation posture,
- route-aware shell behavior,
- and command surfaces that change visible navigation state.

Current entry points:

- `docs/dev/ui/patterns/app-shell.md`
- `docs/dev/ui/patterns/app-shell-patterns.md`
- `docs/dev/ui/patterns/app-structure/search-first.md`
- `docs/dev/ui/patterns/wm-snap.md`
- `docs/dev/ui/navigation/recents.md`

Related DSL contract:

- `docs/dev/dsl/navigation.md`

Rule of thumb:

- if a control affects where the user is in the app rather than just what they edit on the current page, it probably
  belongs in Navigation.

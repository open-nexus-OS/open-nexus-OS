# Input & Selection

This category covers how users enter data, select things, and interact across touch, pointer, keyboard, and IME.

Use it for:

- focus and event routing,
- text input and editing posture,
- gestures,
- IME/candidate handling,
- shortcuts,
- and selection models.

Current entry points:

- `docs/dev/ui/input/input.md`
- `docs/dev/ui/input/text-input.md`
- `docs/dev/ui/input/gestures.md`
- `docs/dev/ui/input/ime.md`
- `docs/dev/ui/input/shortcuts.md`
- `docs/dev/ui/input/spellcheck.md`
- `docs/dev/ui/patterns/transfer-sharing/drag-and-drop.md`
- `docs/dev/ui/patterns/transfer-sharing/clipboard.md`

Rule of thumb:

- if the main question is “how does the user select or enter this?”, start here rather than in Components.

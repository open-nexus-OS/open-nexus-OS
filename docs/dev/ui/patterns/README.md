# UI Patterns

Patterns describe recurring screen/window/app-shell structures.

Use this category for:

- app shell roles and zones,
- windowing, snap, resize, and multiwindow posture,
- lifecycle/window behavior,
- search-first shell guidance,
- and cross-device structural patterns that are larger than a single component.

Rule of thumb:

- if the question is “how should this whole screen/window be organized?” rather than “which control do I use?”, it
  probably belongs in Patterns.

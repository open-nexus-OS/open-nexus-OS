# Status & Feedback

Status docs cover how the system communicates progress, attention, errors, unread state, and other live feedback.

Use this category for:

- progress and loading,
- empty/error states,
- badges and unread state,
- notifications/history-like feedback surfaces,
- and activity/rating/gauge-style status widgets.

Current entry points:

- `docs/dev/ui/status/notifications.md`
- `docs/dev/ui/status/notifications-v2.md`
- `docs/dev/ui/patterns/app-structure/search-first.md`
- `docs/dev/ui/foundations/animation.md`

Rule of thumb:

- if the UI is telling the user “what is happening?” rather than “what can you do?”, it probably belongs here.
